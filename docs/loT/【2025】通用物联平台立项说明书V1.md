# 【通用物联集成平台】立项说明书 V1.0.0

## 一、项目背景与机遇

### 1.1 市场现状

当前智能家居市场呈现“品牌孤岛”态势。各大品牌如小米、华为、海尔等均致力于构建自身的封闭生态，用户购买不同品牌的设备后，需要在多个手机 App 之间切换，操作繁琐，无法实现真正的跨品牌设备联动。这严重破坏了智能家居“无缝联动、主动服务”的核心体验。

### 1.2 竞争对手分析 - 以小米/米家为例

- **优势**：强大的品牌号召力、丰富的自研设备矩阵、成熟的供应链。通过“自研设备+生态链企业+第三方品牌接入”的模式，实现了广泛的设备覆盖。
- **劣势**：
  - **立场偏向性**：其核心是服务自身生态，第三方品牌的功能接入常为“阉割版”，复杂功能仍需打开原品牌 App。
  - **合作门槛**：对于第三方品牌，接入小米生态意味着在一定程度上放弃用户数据、交互入口和部分话语权。

### 1.3 我们的机遇与差异化定位

我们不具备小米的初期硬件优势与行业话语权。因此，我们的核心战略是 **“中立、开放、利他”**。

- **对于用户（C 端）**：我们承诺成为跨品牌场景自动化专家。用户的核心诉求是便捷与智能，而非品牌忠诚度。我们将解决“品牌孤岛”带来的痛点，提供比小米更加全品类的全屋智能体验。
- **对于品牌商（B 端）**：我们并非竞争者，而是服务者与赋能者。我们通过提供广告曝光、匿名用户数据洞察、新增销售渠道和智能化售后服务，帮助品牌商降本增效，实现共赢。我们的平台是它们触达新用户、提升服务价值的增量渠道，而非存量争夺者。

## 二、项目愿景与目标

- **项目愿景**：成为全球最受用户信赖、最受品牌伙伴欢迎的中立性智能家居集成平台。
- **短期目标（1 年内）**：成功上线 MVP 版本，接入 3-5 个主流品牌，实现 2 个核心场景的深度自动化，获取 10,000 名种子用户。
- **中期目标（2-3 年）**：接入超过 50 个品牌，形成完善的自动化场景库与初步的数据服务能力，建立稳定的商业模式。
- **长期目标（3 年以上）**：参与或主导行业标准制定，成为智能家居领域的“数据水源”与“连接标准”，构建强大的品牌护城河。

## 三、核心功能规划与开发步骤

### 第一阶段：最小可行性产品 - 聚焦核心价值（预计周期：4-6 个月）

- **核心目标**：验证“跨品牌自动化”的技术可行性与用户需求。
- **技术路径**：
  - **协议兼容**：优先选择 Wi-Fi 和开放 API 协议的品牌，如飞利浦 Hue、TP-Link（Kasa）、Yeelight 等。技术上，重点构建统一设备抽象层，将不同品牌的 API“翻译”成平台内部的标准化指令。
  - **场景选择**：聚焦 **“回家模式”** 和 **“影院模式”** 。这两个场景涉及灯光、窗帘、影音等设备，能充分体现跨品牌联动的价值。
  - **MVP 功能**：
    - **设备添加**：支持通过账号授权（OAuth 2.0）绑定第三方平台设备。
    - **设备列表**：统一展示所有已绑定的跨品牌设备。
    - **场景创建**：提供图形化界面，让用户可轻松创建“如果...就...”的自动化规则（例如：“如果”工作日晚上 7 点，“就”打开 A 品牌客厅灯、关闭 B 品牌窗帘、打开 C 品牌投影仪）。
- **交付物**：一个界面极其简洁、核心功能流畅的 Android/iOS App。

### 第二阶段：建立合作，丰富生态（预计周期：6-18 个月）

- **核心目标**：扩大设备库，深化自动化能力，启动数据赋能。
- **实现路径**：
  - **案例驱动合作**：利用第一阶段的成功案例和用户好评，主动接触云鲸、大金、美的等品牌的中小型或创新业务部门。向他们展示我们的平台如何为其带来额外用户和销量。
  - **降低接入成本**：为品牌商提供清晰、标准的 SDK 或 API 接入文档，设立“开发者中心”，鼓励它们自主适配。
  - **功能深化**：
    - **自动化升级**：引入更多触发条件（如设备状态、地理位置、时间区间）和执行动作。
    - **数据看板雏形**：在后台为合作品牌提供匿名的、聚合级的设备使用频率、热门场景等数据洞察报告。
    - **UI/UX 优化**：参考米家对云鲸的集成方式，在设备列表页清晰展示设备来源平台，并提供一键跳转至原品牌 App 进行高级设置的入口（如果必要）。
