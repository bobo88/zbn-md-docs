# 云网关智能家居管理平台功能清单说明书

**文档版本**： V1.0 **创建日期**： 2025 年 10 月 27 日 **项目名称**： 云网关智能家居管理平台

## 1. 文档简介

本文档旨在详细定义“云网关智能家居管理平台”项目的全部功能需求。平台旨在为用户提供一个统一、智能、便捷的中心，用于远程管理和联动控制家中的各类智能家电，提供类似小米之家、华为智慧生活、小翼管家等应用的完整体验，平台就像“家庭代理人”一样能高效的对家里的各种电器进行有效管理。本清单将作为项目设计、开发与测试阶段的核心依据。

## 2. 功能模块总览

平台功能主要分为以下六大核心模块：

1. 用户入口与首页
2. 设备管理
3. 智能自动化
4. 我的/个人中心
5. 增值功能
6. 技术后台（支撑层）

## 3. 详细功能清单

红色部分：MVP V1.0.0（用户登录、设备管理、基础场景化）蓝色部分：MVP V2.0.0（设备管理升级版、场景化管理升级版）黄色部分：MVP V3.0.0（实现更多增值功能）

### 3.1. 用户入口与首页

此模块为用户打开应用后看到的主界面，是信息展示与快捷操作的核心入口。

| 一级功能  | 二级功能       | 功能描述                                                         |
| --------- | -------------- | ---------------------------------------------------------------- |
| 用户认证  | 登录/注册      | 支持手机号+验证码、账号密码等多种方式注册与登录。                |
|           | 第三方登录     | 支持微信、Apple ID 等第三方授权快速登录。                        |
|           | 找回密码       | 提供通过手机号或邮箱验证重置密码的流程。                         |
| 首页/总览 | 场景快捷卡片   | 在首页展示用户自定义的场景（如“回家模式”），支持一键执行。       |
|           | 设备状态总览   | 醒目展示在线/离线设备总数，以及关键设备实时状态（如当前温度）。  |
|           | 常用设备控制   | 用户可将常用设备（如客厅灯、空调）固定在首页进行快捷开关和调节。 |
|           | 消息与告警中心 | 显示来自安防设备、系统通知等的最新告警信息，并设有入口。         |
|           | 全局搜索       | 用户可通过搜索框快速查找平台内的设备、场景或功能。               |
|           | 自动化推荐     | 系统根据用户设备使用习惯，智能推荐可创建的自动化规则。           |

### 3.2. 设备管理

此模块是平台的核心，涵盖设备从添加到管理的全生命周期。

| 一级功能 | 二级功能      | 功能描述                                                          |
| -------- | ------------- | ----------------------------------------------------------------- |
| 设备添加 | 自动发现      | 自动扫描并发现局域网内处于待配网状态的智能设备。                  |
|          | 手动添加      | 提供扫描设备二维码、按品类/品牌选择添加等方式。                   |
|          | 配网引导      | 图形化引导用户将设备重置至配网模式，并完成 Wi-Fi 信息配置与连接。 |
|          | 初始化设置    | 添加成功后，引导用户为设备命名并分配至相应房间。                  |
| 设备查看 | 房间视图      | 按家庭的实际房间结构（客厅、卧室等）分类展示设备。                |
|          | 品类视图      | 按设备类型（灯光、家电、安防、环境等）分类展示设备。              |
| 设备控制 | 设备控制面板  | 进入设备详情页后进行远程开关、模式切换、温度/亮度调节等。         |
|          | 实时状态反馈  | 界面实时显示设备的当前工作状态、读数（如功耗、温度）等。          |
| 设备设置 | 设备信息管理  | 支持用户重新命名设备、更换设备图标、移动设备所属房间。            |
|          | 设备共享      | 将设备的控制权限共享给其他家庭成员，并可设置其权限级别。          |
|          | 固件升级(OTA) | 支持手动或自动为设备检查并升级最新固件，提升性能与安全性。        |
|          | 设备移除      | 将设备从平台中解绑并移除，清空相关数据。                          |

### 3.3. 智能自动化

此模块是实现家居“智能化”的关键，让设备之间能够协同工作。

| 一级功能 | 二级功能 | 功能描述 |
| --- | --- | --- |
| 自动化 | 创建自动化 | 触发条件：设备状态（如传感器触发）、定时、地理位置等。执行动作：控制单个/多个设备、执行场景、发送通知。 |
|  | 自动化模板 | 提供丰富的预设自动化模板（如“离家布防”、“起床唤醒”），方便用户快速启用。 |
|  | 自动化管理 | 以列表形式展示所有已创建的自动化，支持单独启用、禁用、编辑或删除。 |
| 场景 | 创建场景 | 将多个设备调整到特定状态后保存为一个场景（如“影院模式”：关灯、降幕布、开投影仪）。 |
|  | 场景执行 | 可通过首页卡片、小组件或语音助手一键触发场景。 |
|  | 场景分享 | 生成场景二维码，其他用户扫描后可一键复制该场景到自己的账户中。 |

### 3.4. 我的/个人中心

此模块用于管理用户账户、家庭设置和系统偏好。

| 一级功能   | 二级功能      | 功能描述                                                   |
| ---------- | ------------- | ---------------------------------------------------------- |
| 家庭管理   | 家庭创建/切换 | 支持用户管理多个家庭（如“我的家”、“父母家”），并自由切换。 |
|            | 成员管理      | 添加或删除家庭成员，并为其分配“管理员”或“普通用户”角色。   |
|            | 房间管理      | 允许用户自定义添加、编辑或删除家庭中的房间。               |
| 账号设置   | 个人资料      | 编辑头像、昵称、手机号、邮箱等个人信息。                   |
|            | 账号与安全    | 修改密码、管理绑定的第三方账号、登录设备管理等。           |
|            | 消息设置      | 允许用户自定义各类推送通知（如设备告警、营销信息）的开关。 |
| 系统与帮助 | 意见反馈      | 用户可在此提交使用中遇到的问题或改进建议，并可上传截图。   |
|            | 帮助中心      | 集成常见问题解答（FAQ），帮助用户自助解决常见问题。        |
|            | 关于我们      | 展示应用版本号、服务协议、隐私政策等必要信息。             |

### 3.5. 增值功能

此模块为提升用户体验和平台商业价值的扩展功能。

| 功能模块 | 核心功能点 | 功能描述 |
| --- | --- | --- |
| 能耗统计 | 用电统计 | 以日、周、月、年为单位，统计家庭总耗电量及单个支持设备的耗电量。 |
|  | 数据可视化 | 以图表形式展示能耗数据变化趋势，并生成简单的能耗报告。 |
| 安防中心 | 布防/撤防 | 一键启用或关闭所有安防设备（如门窗传感器、motion 传感器）的告警功能。 |
|  | 告警日志 | 集中查看所有安防相关的历史触发记录，包括时间、设备名称和类型。 |
| 云服务 | 云存储服务 | 为智能摄像头提供云端录像存储与回放功能，通常作为付费订阅服务。（设备有内存卡的情况下支持免费回放功能） |

### 3.6. 技术后台（支撑层）

此部分为用户不可见的后端系统，是平台稳定运行的基石。

| 功能领域 | 核心要求 | 描述 |
| --- | --- | --- |
| 设备接入层 | 多协议支持 | 需支持主流物联网通信协议，如 Wi-Fi, Bluetooth Mesh, Zigbee, Z-Wave 等。 |
| 平台安全 | 设备认证与加密 | 确保设备与云端、App 与云端之间的通信经过严格认证和数据加密。 |
| 服务架构 | 高可用与弹性伸缩 | 平台架构需保证高可用性，并能根据用户量动态伸缩，应对访问压力。 |
| 数据处理 | 规则引擎 | 用于解析和执行用户创建的所有自动化场景和联动规则。 |
|  | 实时消息推送 | 保证设备状态变化和用户指令能够低延迟地推送到 App 客户端。 |
|  | 数据存储与分析 | 存储设备状态、用户操作日志等海量数据，并为大数据分析和 AI 推荐提供支持。 |

## 4. 版本规划建议

- **V1.0 MVP (最小可行产品)**
  - **核心功能**：重点保证设备添加、控制和家庭管理的流畅与稳定。
  - **智能核心**：实现 3.3 中基础的“自动化”与“场景”创建功能。
- **V2.0 体验优化**
  - **功能完善**：逐步实现各模块中的中优先级功能，如自动化模板、能耗统计、安防中心等。
  - **性能提升**：优化系统性能与响应速度，增强稳定性。
- **V3.0 增值与生态**
  - **商业探索**：推出云存储等付费增值服务。
  - **生态扩展**：开放第三方设备接入平台，引入更多品牌和品类，构建生态系统。
  - **AI 赋能**：深化基于用户习惯的智能化推荐和自学习自动化。
