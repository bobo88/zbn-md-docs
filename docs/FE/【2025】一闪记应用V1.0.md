# 一闪记应用（Version 版本迭代清单）

本应用的目标是提供极简、快速的文本记录功能，并逐步迭代为一款智能知识管理工具，最终能够自动整理记录，形成结构化内容，以及可发布为文章。

## V0.1 - MVP 版本（最小可行产品）

支持极速记录、自动保存、本地存储，并可选（需要登录）云端同步的极简笔记应用。

### 1. 核心目标

1. **极速记录：** 无需复杂操作，打开即用，记录文本。
2. **存储闭环：** 用户能够创建、编辑、删除文本，并持久化存储。（本地存储）
3. **跨设备可用：** 可通过简单方式在不同设备间访问笔记。（需要用户登录）

### 2. 功能清单

#### 2.1 基础功能

- [ ] **无感知使用：** 打开网页即用，无需注册即可记录。
- [ ] **文本记录：** 纯文本输入，支持快捷键提交。
- [ ] **自动保存：** 输入内容自动保存在本地存储（localStorage），默认按照时间戳命名。
- [ ] **云端存储 API：** 提供后端 API，支持远程存储。（需要用户登录）
- [ ] **简单列表管理：** 按时间排序展示记录内容。
- [ ] **快速搜索：** 本地搜索笔记标题或部分内容。
- [ ] **导出功能：** 支持一键导出全部笔记（TXT/Markdown）。

#### 2.2 存储机制

- [ ] **用户唯一目录：** 后端为每个用户创建专属目录。（需要用户登录）
- [ ] **日期归档：** 按年月日自动分类笔记存储。（需要用户登录）

#### 2.3 设备兼容

- [ ] **适配移动端 & PC 端：** 网页自适应，适合触屏和键盘操作。

### 3. 关键技术节点

#### 3.1 前端

##### 3.1.1 轻量级 UI 方案

- 快速打开，无需加载等待

##### 3.1.2 本地存储和搜索方案

- **本地存储方案：**
  - 使用 localStorage 存储文本，自动命名（时间戳）
  - 采用 IndexedDB 进行数据管理（后续优化）
- **前端模糊搜索：** 基于 `indexOf()` / `includes()` 进行关键词匹配

##### 3.1.3 搜索优化：

- Web Worker 处理搜索逻辑，避免 UI 卡顿

##### 3.1.4 时间排序：

- 以 timestamp 作为排序字段
