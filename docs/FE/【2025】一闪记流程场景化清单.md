# 一闪记 - 流程场景化清单

**日期：** 2025.04.03

一闪记应用的核心应用场景及流程，涵盖**笔记创建、管理、分类、同步、共享、AI 辅助、社交互动**等关键功能。

**核心场景：**

1. **笔记创建与管理**（最基本的核心功能）；
2. **笔记多端同步**（笔记与用户强相关绑定）；
3. **AI 智能辅助**（提速笔记的质量和完成效率）。

## 1. 笔记创建与管理（核心场景）

**目标：** 用户可以高效创建、编辑、保存和管理笔记。

### 1.1 创建笔记

**📌 流程：**

1. 用户进入笔记创建页面。
2. 输入笔记标题、内容，可选择标签和笔记本。
3. 底部固定**保存按钮**，用户点击后触发 `saveNote()` 方法。
4. API 调用，返回成功后跳转到**笔记列表页**。

**📌 关键交互：**

- **自动保存（可选）：** 输入笔记时，定期自动存储草稿。
- **加载状态 & 提示信息：** 保存前后提供状态反馈。
- **避免按钮遮挡内容：** 增加 `padding-bottom` 适配底部按钮。

### 1.2 笔记列表

**📌 流程：**

1. 进入笔记列表页，默认展示所有笔记。
2. 用户可通过**搜索框、标签、笔记本、时间**进行筛选。
3. **左滑笔记**，显示**编辑、删除**按钮。
4. **点击笔记**进入编辑页面，传递 `noteID` 加载笔记内容。

**📌 关键交互：**

- **搜索 & 筛选：** 支持按关键字、标签、时间筛选笔记。
- **编辑：** 点击笔记可直接进入编辑器。
- **删除确认：** 防止误删，提供二次确认弹窗。

## 2. 分类与标签

**目标：** 帮助用户高效管理笔记，提高查找效率。

**📌 流程：**

1. 用户进入**分类管理页面**，查看已有**笔记本 & 标签**。
2. 支持**添加、删除**笔记本/标签（删除前有确认提示）。
3. 选择标签，可筛选出所有关联笔记。

**📌 关键交互：**

- **模拟数据展示：** 初期无后端 API 时，使用假数据演示交互。
- **交互优化：** 删除笔记本时，需确认是否删除所有关联笔记。

## 3. 同步管理（核心场景）

**目标：** 确保笔记数据在不同设备间同步。

**📌 流程：**

1. 进入**同步管理页面**，显示**当前同步状态、最后同步时间、设备数**。
2. 用户（需登录）可开启**自动同步**，设置同步周期。
3. **设备管理：**
   - 查看已同步设备状态（在线/离线）。
   - 手动移除某个设备的同步权限。
4. **手动同步功能**，一键同步所有笔记数据。
5. 提供**清除本地缓存**选项。

**📌 关键交互：**

- **同步状态提示：** 明确告知用户同步进度。
- **缓存清理：** 提供二次确认，避免误删数据。

## 4. 共享与协作

**目标：** 支持用户分享笔记并进行权限管理。

**📌 流程：**

1. 用户在笔记详情页点击 **“分享”** 按钮。
2. 进入**分享设置**，选择：
   - 只读 / 可编辑权限。
   - 生成分享链接，一键复制。
3. 用户可在**已分享列表**查看并管理分享笔记：
   - 取消分享。
   - 更新权限（改为只读/可编辑）。

**📌 关键交互：**

- **分享权限控制：** 允许笔记拥有者修改权限。
- **快捷复制：** 支持一键复制分享链接。

## 5. AI 智能辅助（核心场景）

**目标：** 利用 AI 提高笔记管理效率。

**📌 流程：**

1. 用户输入笔记内容后，点击**AI 智能标注**。
2. 系统自动分析内容，生成推荐标签并展示。
3. 用户可以选择接受或修改标签。
4. **相关笔记推荐：**
   - 选择目标笔记后，AI 分析相似笔记并推荐。
   - 以列表形式展示推荐笔记。

**📌 关键交互：**

- **AI 处理状态：** 生成标签和推荐笔记时显示加载状态。
- **用户可编辑：** AI 生成的标签支持手动修改。

## 6. 高级设置

**目标：** 提供定制化的使用体验。

**📌 流程：**

1. 用户进入**设置页面**，管理：
   - **提醒设置：** 开启/关闭提醒、设置提醒时间、重复周期。
   - **待办事项：** 新增/删除/标记完成。
   - **离线模式：** 开启后可在无网状态下编辑笔记，稍后自动同步。
2. 选择对应选项，修改并保存设置。

**📌 关键交互：**

- **待办事项操作：** 快速添加、完成、删除待办。
- **离线模式：** 告知用户哪些功能可用/不可用。

## 7. 我的收藏 & 社交互动

**目标：** 打造笔记社区，增加互动性。

**📌 流程：**

1. **收藏笔记：**
   - 在笔记详情页点击**收藏**按钮。
   - 收藏的笔记可在 **“我的收藏”** 页面查看。
2. **社交互动：**
   - 笔记支持**点赞、评论**。
   - 可**关注作者**，查看其发布的更多内容。
3. 在 **“我的发布”** 页面，用户可：
   - 查看自己发布的笔记及互动情况（浏览量、评论数）。
   - 切换笔记的**公开/隐藏**状态。

**📌 关键交互：**

- **实时更新点赞/评论**，无需刷新页面。
- **关注系统**，让用户获取感兴趣的内容。

## 8. 排行榜

**目标：** 增加用户激励，提升活跃度。

**📌 流程：**

1. 用户进入**排行榜页面**，查看：
   - **热门笔记**排行。
   - **活跃用户**排行（根据笔记数量、点赞、收藏等）。
2. **前三名特别标注**（金银铜牌）。
3. 点击用户可查看其**个人主页**及笔记。

**📌 关键交互：**

- **数据定期更新**，保持竞争性。
- **提供激励**，如奖励徽章、成就系统等。
