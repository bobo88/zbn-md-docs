# 前端核心项目技术文档

## 一、项目概述

本项目为公司核心前端项目，采用现代化前端技术栈进行开发，具备良好的组件化、模块化结构，便于多人协作、维护和二次开发。

- **项目名称**：CMS 管理后台系统（前端项目）
- **技术栈**：Vue 3 + Vite + TypeScript + Pinia + Element Plus
- **目标平台**：PC / 移动端（响应式）
- **项目地址**：https://gl.szzbn.cn/FE/cms-vue3

## 二、项目结构说明

```
├── public/               # 公共静态资源
├── src/                  # 核心源码目录
│   ├── assets/           # 静态资源：图片、样式等
│   ├── components/       # 通用组件
│   ├── views/            # 页面级组件（按路由划分）
│   ├── layouts/          # 布局组件
│   ├── router/           # 路由配置
│   ├── store/            # 状态管理（Pinia）
│   ├── utils/            # 工具函数
│   ├── composables/      # 组合式函数（Vue 3 专属）
│   ├── types/            # TypeScript 类型定义
│   ├── services/         # 请求封装（axios）
│   └── App.vue           # 应用入口
├── .env.development      # 开发环境变量
├── .env.production       # 生产环境变量
├── vite.config.ts        # Vite 配置文件
└── package.json          # 项目依赖与脚本
```

## 四、书写规范

### 1. 目录规范

- 所有文件和文件夹命名统一使用 **小写+中划线（kebab-case）**
- 页面组件放在 `views/`，组件放在 `components/`，并保持目录整洁

### 2. 命名规范

- **组件命名**：`XxxComponent.vue`
- **变量命名**：使用 `camelCase`
- **枚举常量**：使用 `SCREAMING_SNAKE_CASE`

### 3. 代码风格

- 使用 ESLint + Prettier 强制统一代码格式
- 统一使用双引号 `" "`，缩进为 2 空格
- 单文件组件顺序：`<template>` → `<script setup>` → `<style scoped>`

### 4. 提交规范（建议使用）

配合 `commitlint` + `husky` + `lint-staged`，强制提交格式：

```bash
# 示例
feat: 新增用户登录功能
fix: 修复注册时验证码不显示问题
refactor: 重构表单组件
```

## 五、打包与运行命令

```bash
# 安装依赖
pnpm install

# 本地开发环境
pnpm dev

# 打包生产环境
pnpm build

# 本地预览构建效果
pnpm preview

# 格式化代码
pnpm lint
```

## 六、环境变量说明

在 `.env.*` 文件中配置以下变量：

```
VITE_API_BASE_URL=https://api.example.com
VITE_APP_TITLE=XXX系统
```

> **说明**：所有环境变量都需要以 `VITE_` 前缀开头，才能在代码中通过 `import.meta.env` 访问。

## 七、注意事项

1. 路径别名统一使用 `@` 表示 `src` 根目录
2. 所有请求接口通过封装好的 `axios` 实例进行调用，不允许直接使用 `fetch`
3. 统一使用组件库的表单、弹窗、提示等，避免原生 DOM 操作
4. 状态管理统一使用 `Pinia`，避免多源状态造成维护困难
5. **严禁**将密码、Token 等敏感信息写入代码或 `.env` 文件中，必须使用后端下发机制

## 八、开发建议

- 每个页面模块建议以 **“页面入口 + 内部组件”** 的方式组织，利于拆分维护
- 多人协作时请遵守代码规范并提前沟通好模块划分
- 公共样式、组件、hooks 使用前请先查阅是否已存在，避免重复造轮子

## 九、常见问题 FAQ

| 问题           | 解决方式                                     |
| -------------- | -------------------------------------------- |
| 页面刷新后空白 | 检查路由 `history` 模式和 `nginx` 配置       |
| 跨域问题       | 开发阶段使用 Vite 的 `proxy`，生产由网关处理 |
| 图标不显示     | 检查图标库是否正确引入、路径是否正确         |
